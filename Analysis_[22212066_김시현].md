## 1. Introduction (도입)

### 1.1 Summary

* ** 복잡한 시스템과 긴 플레이 타임을 요구하는 하드코어 게임에서 벗어나, 현대인들이 짧은 시간 동안 직관적으로 즐길 수 있는 스낵 컬처형 게임 <b>‘Bomb-Tag(나만 아니면 돼~)’</b>이다.**

### 1.2 게임소개 (Game Introduction)

* **‘Bomb-Tag(나만 아니면 돼~)’는 오프라인의 ‘폭탄 돌리기’ 규칙을 온라인 실시간 멀티플레이어 환경으로 이식한 하이퍼 캐주얼 파티 게임이다.**

* **핵심 메커니즘: 제한 시간 내에 자신의 폭탄을 다른 플레이어에게 넘겨야 하는 원초적인 긴장감을 제공하며, 실시간 추격과 충돌이라는 게임의 본질적인 재미에 집중한다.**

* **활용도: 지인들과의 가벼운 내기나 모임에서의 아이스브레이킹을 위한 매개체로 활용될 수 있도록 설계되었다.**

### 1.3 Goals

* **실시간 네트워크 환경 구축: 플레이어 간의 이동 위치와 폭탄 소유 상태가 지연 없이 동기화되는 안정적인 멀티플레이 환경을 구현한다.**

* **직관적이고 빠른 템포의 게임성 제공: 별도의 학습 없이 즉시 플레이할 수 있도록 규칙을 단순화하고, 3~5분 내외의 짧은 플레이 타임을 통해 속도감 있는 경험을 제공한다.**

* **공정하고 정확한 판정 시스템: 서버 권한(Server Authority)을 기반으로 플레이어 간의 충돌(폭탄 양도) 및 제한 시간 만료 시점의 판정을 정확하게 처리하여 승패의 신뢰도를 확보한다.**

* **서버(Server) 액터 생략의 이유:**

본 프로젝트는 언리얼 엔진 5의 리슨 서버(Listen Server) 아키텍처를 채택하고 있다. 호스트 플레이어의 클라이언트가 곧 서버 역할을 겸하며, 동기화 및 판정 로직은 외부 시스템이 아닌 개발 대상인 시스템(System Boundary) 내부의 핵심 로직(GameMode)으로 간주된다. 따라서 서버를 외부 조력자(Secondary Actor)로 분리하지 않고 시스템 내부 기능으로 통합하여 표현했다.

* **Include 관계의 최소화:**

방 참여나 게임 시작, 캐릭터 이동 등의 기능은 논리적으로 '로그인(UC1)'을 거쳐야만 수행할 수 있다. 하지만 모든 기능에 `<<include>>`화살표를 연결하면 다이어그램의 가독성이 크게 떨어진다. 따라서 로비 진입의 필수 관문인 '방 만들기/참여하기'에만 Include 관계를 명시하고, 인게임 기능(UC6~UC9)은 각 유스케이스 명세서의 '사전 조건(Preconditions)'에 명시하는 방식으로 복잡도를 낮췄다.

### 3.2 Domain Class Description

1. ### Player (플레이어)

* 시스템에 접속한 사용자를 나타내는 기본 데이터 클래스이다. 사용자의 식별자인 Nickname과 게스트 여부(IsGuest), 호스트 권한 여부(IsHost)를 가진다. 

2. ### LobbySystem (로비 시스템)

* 게임 시작 전 매치메이킹을 관리한다. 현재 개설된 방(RoomList)을 관리하며, 사용자의 요청에 따라 방 생성(CreateRoom) 및 참여(JoinRoom)를 처리한다.

3. ### GameSession (게임 세션)

* 개별 게임 방을 의미하며, 방 이름, 최대 수용 인원(MaxPlayers) 및 현재 접속 인원 등의 메타데이터를 유지하여 네트워크 세션을 관리한다.

4. ### GameMap (게임 맵)

* 플레이어들이 활동하는 가상의 아레나 공간이다. 라운드가 시작될 때 플레이어와 아이템이 생성될 무작위 스폰 좌표(SpawnPoints)를 관리한다.

5. ### BombTag_GameMode (게임 모드 - 서버 전용)

* 게임의 핵심 룰(Rule)을 주관하는 최고 권한 클래스이다. 리슨 서버(Host)에서만 구동되며, 플레이어 간의 충돌 검증(CheckTagCollision), 아이템 획득 유효성 검증(ValidateItemAcquisition), 라운드 종료(EndRound) 등 중요한 판정을 내린다.

6. ### BombTag_GameState (게임 상태 - 전체 동기화)

* GameMode가 결정한 게임의 현재 상태(남은 시간, 생존자 수, 현재 폭탄 소유자 등)를 모든 클라이언트(Client)에게 실시간으로 동기화(Replication)하여 화면(UI)에 표시할 수 있게 해주는 데이터 클래스이다.

7. ### PlayerController (플레이어 컨트롤러)

* 사용자의 물리적 입력(키보드/마우스)을 받아들이는 인터페이스입니다. 네트워크 환경을 고려하여 서버로 이동 신호(SendMoveInput) 및 태그/획득 상호작용 신호를 전송한다.

8. ### Character (캐릭터 폰)

* 인게임에서 플레이어의 입력을 받아 움직이는 시각적 아바타(Pawn)입니다. 현재 위치(Location)와 생존 여부(IsAlive) 상태를 가진다.

9. ### Bomb (폭탄)

* 게임의 승패를 가르는 핵심 오브젝트이다. 폭발 타이머(TimeLeft)를 가지며, 특정 캐릭터에 부착되어 있다가 시간이 만료되면 폭발(Explode)하여 해당 캐릭터를 탈락시킨다.

10. ### Item (아이템)

* 게임의 변수를 창출하는 오브젝트이다. 이동 속도 증가 등 고유의 효과(ItemType)를 가지고 있다. 인벤토리에 저장되지 않고 캐릭터와 충돌하는 즉시 소비(Instantly Consumes)되며, 지정된 시간(Duration) 동안 즉각적인 버프 효과를 적용(ApplyInstantEffect)한다.
